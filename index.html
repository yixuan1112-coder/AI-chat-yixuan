Split Protocols

Random Split (80/10/10)
Cycles randomly shuffled across train/val/test. Simplest baseline — may overfit to temporal patterns.
Temporal Split
Early cycles → train, middle → val, late → test. Tests true forecasting capability under distribution shift.
Cross-Cell Split
Entire cells held out for test. Evaluates generalization to unseen battery units with different degradation trajectories.

Technical reference for using the BatteryTwin Benchmark datasets and evaluation framework.

Getting Started

Clone the repository and install the conda environment:

git clone https://github.com/tianwen1209/BatteryTwin-Benchmark-DataPrep.git
cd BatteryTwin-Benchmark-DataPrep
conda create -n batterytwin python=3.10
conda activate batterytwin
pip install -r requirements.txt

Dataset Registry

All datasets are tracked in dataset_registry.csv with columns: dataset_id, dataset_name, data_category, doi, source_url, assigned_student, ref_name, status, metadata_done, timeseries_done, cycle_summary_done, dataset_note_done, qc_done, last_updated, notes.

Running ETL

Each dataset has a dedicated ETL script in scripts/ following the naming pattern etl_dataset_XX.py. Run with:

python scripts/etl_dataset_01.py --input raw/dataset_01/ --output processed/dataset_01/

Evaluation

The benchmark evaluation framework uses evaluate.py and dataset_interface.py to run all five baseline models across selected datasets and split protocols.

About

BatteryTwin Benchmark is a comprehensive benchmark platform for lithium-ion battery prognostics and health management (PHM). The project curates, standardizes, and evaluates battery cycling datasets from research labs worldwide, enabling fair comparison of machine learning models for state-of-health estimation and remaining useful life prediction.

Quality Assessment & Diagnostics
Every dataset in BatteryTwin is evaluated across four key quality dimensions to ensure scientific reliability. Assessments include physical plausibility checks and detailed JSON diagnostic reports.
0–1
Completeness
Data coverage and availability across all cycles and measurement channels.
0–1
Consistency
Internal logical consistency: monotonic timestamps, cycle numbering, and pattern adherence.
0–1
Accuracy
Physical plausibility: voltage range, energy balance, coulombic efficiency bounds.
0–1
Validity
Schema compliance, column type correctness, required field presence, value range validity.
Physical Plausibility Checks
  • ✓
    Voltage Range Validation
    All cell voltages within 2.0V–4.5V nominal operating range for the stated chemistry.
  • ✓
    Energy Balance Check
    Charge/discharge energy integral consistency — coulombic efficiency 95–105% per cycle.
  • ✓
    Capacity Monotonicity
    Degradation trajectory follows expected non-increasing trend with allowable recovery windows.
  • !
    Temperature Consistency
    Cell surface temperature within ±3°C of stated test condition; flags thermal runaway events.
  • ✓
    Timestamp Integrity
    Monotonically increasing timestamps with no negative intervals or unreasonable gaps (>24h).
  • ✓
    Current Direction Consistency
    Charge current positive, discharge current negative (or vice versa) — consistent sign convention throughout.
Diagnostic Output Format

Quality diagnostics are exported as structured JSON files per dataset, enabling programmatic integration into downstream pipelines.

{
  "dataset_id": "dataset_03",

4
Publication
Your dataset is added to the catalog with full provenance, DOI, and your attribution.
What to Submit
Raw Data Files
Time-series measurements in CSV, Parquet, HDF5, MATLAB, or native cycler format (Arbin, MACCOR, Neware).
Metadata
Cell chemistry, nominal capacity, form factor, temperature, C-rate protocol, number of cells, and cell identifiers.
Protocol Description
Test protocol document detailing charge/discharge procedures, rest periods, characterization tests, and equipment used.
License / DOI / Source
Publication DOI, data license (CC-BY preferred), original source URL, and any attribution requirements.
