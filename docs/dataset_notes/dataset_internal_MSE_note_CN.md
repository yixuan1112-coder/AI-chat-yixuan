
# NTU_MSE Internal Battery Dataset - 数据整理说明 (README)

## 1. 数据集概况

* **Dataset ID**: `dataset_internal_MSE`
* **原始数据规模**: 包含 167 组真实的物理电池文件夹（如 `1.7AhLFP_001`, `3.3AhNCA_001` 等）。
* **电池规格**: 涵盖 18650 圆柱形电芯，典型容量包括 1.7Ah (LFP) 与 3.3Ah (NCA) 等，额定电压随材料体系分布。
* **材料体系**: 多体系混合，主要包括 LFP (磷酸铁锂) 与 NCA (镍钴铝)，负极均为 Graphite (石墨)。
* **包含文件**:
    1.  `*_metadata.csv` (元数据表，严格对齐 XJTU 18 列标准)
    2.  `*_cycle_summary.csv` (周期汇总表，对齐 XJTU 14 列标准)
    3.  `*_timeseries.csv` (时间序列表，对齐 9 列标准特征)

## 2. 字段映射与物理量标准化

根据 `BatteryTwin Schema v0.2` 规范，对原始数据进行了以下标准化处理：

* **物理常识补全与映射**: 原始 JSON 中仅提供简写的材料标签（如 "LFP"），脚本在处理元数据时内置了物理字典，自动将其补全为标准化学式（如 `LiFePO4`），并根据电化学特性自动映射标称电压（LFP 对应 3.2V，NCA 对应 3.6V）。
* **唯一化标识 (Cell ID 隔离机制)**: 为确保在合并多数据集时 ID 不冲突，`cell_id` 统一采用 `{Dataset_ID}_{Folder_Name}` 格式（例：`dataset_internal_MSE_1.7AhLFP_001`）。但在文件名存储上，为了保持路径简洁，移除了前缀。
* **充放电协议标准化**: 原始工况记录为 `1C CC` 等非规范格式。清洗逻辑通过正则表达式提取倍率，并结合 JSON 中的截止电压，重构为 XJTU 风格的描述性文本（例：`1C CC to 3.65V`）。
* **浮点数精度截断**: 针对时序数据中常见的二进制转换误差，导出时采用了 `%g` 智能格式化，消除了长尾小数（如 `3.60000000002` 截断为 `3.6`），在不损失精度的前提下大幅降低了存储占用。

## 3. 核心工程挑战与清洗逻辑记录

*注：由于内部数据集来源设备多样且文件编码不统一，本数据集的 ETL (抽取、转换、加载) 过程针对性地解决了以下工程痛点：*

1.  **多重字符编码冲突的自动化解决**
    在处理 `3.3AhNCA` 等特定批次数据时，由于原始 CSV 包含特殊字符（如 `℃` 或中文备注），导致标准 `UTF-8` 解码失败并报 `0xa6` 错误。为此，ETL 脚本构建了基于 `Try-Except` 的**智能编码探测架构**，按 `UTF-8 -> GBK -> GB18030 -> Latin1` 的顺序自动切换翻译器，确保了 167 个电池文件 100% 自动读取成功。
2.  **非结构化 JSON 到 18 列宽表的动态转换**
    原始 `metadata.json` 采用了嵌套字典结构（如环境参数嵌套在 `environment_profile` 下）。脚本实现了一套动态解析逻辑，能够穿透嵌套层级提取 `ambient_temp_c` 等关键字段，并对 `form_factor` 字段执行了**智能拆词算法**（将 `cylindrical_18650` 自动拆分为形状与尺寸两列），实现了与 XJTU 标准元数据表的一键对齐。
3.  **超大规模时序数据的“瘦身”与分发**
    考虑到单电池时序数据体积巨大，采用了**“1:1 同名目录镜像分发”**模式，将处理后的特征数据精准存入对应的 167 个子文件夹中。在提取过程中，脚本对 `Step Type` 执行了模糊匹配清洗（将 `CC DChg` 等原始标签统一映射为标准 `discharge`），极大地方便了后续基于特定工况的特征提取工作。
