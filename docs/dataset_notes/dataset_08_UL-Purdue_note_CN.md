# UL-PUR Battery Dataset - 数据整理说明 (README)

## 1. 数据集概况

* **Dataset ID**: `dataset_08`
* **原始数据规模**: 涵盖多种测试温度（如 23°C, 25°C）和不同倍率工况的电池老化数据。
* **电池规格**: 同时包含 18650 圆柱形电芯 (容量约 3.4Ah) 和 Pouch 软包电芯，额定电压 3.6V。
* **材料体系**: NCA (正极: LiNiCoAlO2, 负极: Graphite)
* **包含文件**:
1. `dataset_08_metadata.csv` (元数据表，18 列 XJTU 标准)
2. `*_cycle_summary.csv` (各电池周期汇总表)
3. `*_timeseries.csv` (各电池时间序列表)

## 2. 字段映射与物理量标准化

根据 `BatteryTwin Schema v0.2` 规范，对原始数据进行了以下标准化处理：

* **封装形态区分**: 针对该数据集混合封装的特性，元数据提取脚本通过扫描文件名，成功区分并映射了 `model_or_size` (18650 / Pouch) 与 `form_factor` (cylindrical / pouch) 字段。
* **浮点数精度截断过滤 (长尾小数消除)**: 原始底层二进制数据在 Pandas 导出时会产生无效的浮点数误差（如 `3.6699998...`）。为还原测试设备精度并优化文件体积，在导出 `to_csv` 阶段强制套用了智能格式化 (`float_format='%g'`)，保持了输出数据的纯净外观。
* **温度特征聚合**: 原始汇总表不包含温度数据。已通过读取高频时序文件，利用 `groupby` 按周期计算提取了该循环的 `temperature_max_C` 和 `temperature_avg_C`。

## 3. 核心工程挑战与清洗逻辑记录

*注：UL-PUR 数据集在列名命名和实验末期数据质量上存在特异性，已通过定制化 Python 脚本解决以下工程痛点：*

1. **"Date_Time" 文本列对时间计算的严重干扰**
在计算时序时间差时，遇到了 `unsupported operand type(s) for -: 'str' and 'str'` 的致命报错。排查发现，原始 CSV 的第一列为包含日期的字符串 `Date_Time`，被常规的 `find_col` 错误识别为时间特征。随后重构了 `find_col` 函数的正则表达式，加入了**“防御性跳过逻辑”**（遇到包含 `date` 字样的列直接 `continue`），从而精准抓取到纯数字格式的累积时间列 `Test_Time`，彻底修复了 NaN 和计算报错问题。
2. **寿命末期异常零容量长尾的清洗**
部分电池在完全失效或测试设备断电后，仪器仍会继续记录大量容量为空或接近 0 的无效循环，导致 SOH 曲线严重失真。为此，在数据清洗管线中加入了连续零值检测与过滤算法 (`KEEP_ZERO_N = 3`)。该算法能够智能截断末尾的“死循环”，仅保留前 3 个零容量点作为寿命终结（EoL）的边界信号，极大提高了下游特征工程的信噪比。
3. **特异性列名的模糊匹配**
不同测试柜导出的列名存在细微差异（例如 `Cell_Temperature`, `Temp`, `Environ` 等）。脚本构建了基于优先级顺序的关键词匹配数组，实现了对所有有效数值列（电压、电流、容量、温度、周期索引）的鲁棒性抓取。
