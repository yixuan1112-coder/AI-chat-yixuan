# HUST 电池数据集 - 数据处理说明 (README)

## 1. 数据集概览
* **数据集编号 (Dataset ID)**: `dataset_41`
* **原始数据范围**: 多个 Python Pickle (`.pkl`) 文件，按电池编号（如 `1-1`, `2-5`）组织。
* **电池规格**: A123 Systems APR18650M1A 圆柱电池，额定容量 1.1Ah，额定电压 3.3V。
* **化学体系**: LFP (磷酸铁锂)。
* **产出文件**:
  1. `dataset_41_HUST_metadata.csv` (元数据表)
  2. `HUST_{Cell_ID}_cycle_summary.csv` (循环级汇总表)
  3. `HUST_{Cell_ID}_timeseries.csv` (高频时序表)

## 2. 字段映射与标准化
严格遵循 `BatteryTwin Schema v0.2` 规范进行标准化：
* **单位对齐**: 
    - 原始电流 (`mA`) 统一除以 1000 转换为安培 (`current_A`)。
    - 原始容量 (`mAh`) 统一转换为安时 (`capacity_Ah`)。
* **电流符号对齐**: 严格执行“放电为负”原则。脚本自动处理原始电流信号，确保放电阶段数值为负。
* **健康状态 (SOH) 计算**: 通过公式 $\text{SOH} = \frac{\text{放电容量}}{\text{额定容量 (1.1Ah)}}$ 自动生成 SOH 指标。
* **字典结构扁平化**: 原始数据高度嵌套在 Python 字典中。脚本通过递归遍历将其展平为符合 Schema 的关系型表格，并保持严格的 `cycle_id` 索引。

## 3. 核心工程挑战与流水线逻辑
*备注：HUST 数据集处理的核心在于序列化对象的解析以及从嵌套 DataFrame 中重构循环特征。*

1. **Pickle 对象的深度遍历与反序列化**
   不同于 MAT 或 CSV 格式，HUST 数据以 Python Pickle 序列化对象存储。ETL 流水线通过对 `rul`、`dq` 和 `data` 等键的结构化遍历，确保从跨越数千个循环的复杂对象中提取出一致的物理特征。
2. **嵌套 DataFrame 的片段特征提取**
   虽然总放电容量在字典根节点直接给出，但具体的循环特征（如充放电时长）被包裹在 `data` 键下数千个独立的 Pandas DataFrame 中。脚本通过迭代这些子对象，精确计算出每个循环的时间维度指标。
3. **全生命周期数据同步**
   Pipeline 确保了汇总表中的 `cycle_id` 与高频时序 CSV 中的循环索引完全对应。这种对齐对于下游的 RUL（剩余寿命）预测等机器学习任务至关重要。

## 4. 缺失字段说明
受原始数据局限，以下字段为空：
* `temperature_C` (时序表): 提供的 `.pkl` 数据帧中缺少高频温度采样。
* `internal_resistance_Ohm` (汇总表): 该版本数据集中未显式记录每个循环的内阻测量值。
