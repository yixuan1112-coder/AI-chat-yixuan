
# 重庆大学 (CQU) 电池数据集 - 数据处理说明 (README)

## 1. 数据集概览
* **数据集编号 (Dataset ID)**: `dataset_39`
* **原始数据范围**: 包含 `Dataset1_Dataset2.mat`、`Dataset3.mat`、`Dataset4.mat` 等多个打包文件。
* **电池规格**: 三星 (Samsung SDI) INR18650-20R 圆柱电池，额定容量 2.0Ah，额定电压 3.6V。
* **化学体系**: NMC (正极: LiNiMnCoO2, 负极: 石墨)。
* **产出文件**:
  1. `dataset_39_CQU_metadata.csv` (元数据表)
  2. `CQU_{Cell_ID}_cycle_summary.csv` (循环级特征表)
  3. `CQU_{Cell_ID}_timeseries.csv` (高频时序表)

## 2. 字段映射与标准化
严格遵循 `BatteryTwin Schema v0.2` 规范进行标准化：
* **单位转换与归一化**: 
    - 原始汇总数据中的 `Capacity` 以百分比 (%) 存储，脚本通过 `(val / 100.0) * 2.0` 公式将其转换为标准安时 (`capacity_Ah`)。
    - 时序数据中的毫安 (`mA`) 均除以 1000 转换为安培 (`current_A`)。
* **电流符号对齐**: 严格执行“放电为负”原则。脚本自动确保放电阶段 `current_A` 为负值。
* **唯一标识符 (Cell ID)**: 由于 CQU 数据集将 30 块电池打包在同一个 `.mat` 文件中，脚本从内部 `cell` 变量提取原始编号并统一添加前缀生成全局唯一 ID（如 `CQU_C1`）。

## 3. 核心工程挑战与流水线逻辑
*备注：由于 CQU 数据集采用了复杂的 MATLAB v7.3 嵌套格式，其 ETL 过程涉及深度指针解包。*

1. **MATLAB v7.3 (HDF5) 兼容性瓶颈**
   原始文件为 `v7.3` 格式，无法使用常规方法读取。Pipeline 引入了 `h5py` 库，将 `.mat` 文件视为 HDF5 数据库进行层级遍历，解决了新版 MATLAB 文件的读取难题。
2. **多级指针解包与递归钻取**
   数据结构极深（顶层变量 -> Workingprofile 指针列表 -> 片段引用 -> 最终矩阵）。脚本实现了一套**递归解包逻辑**，穿透引用列表（Cell Array），自动解开指针链获取最底层数值采样点。
3. **时序片段的垂直缝合**
   单块电池的时序数据被拆分为多个片段（如 14 个片段）存储。脚本在钻取到每段数据后使用 `numpy.concatenate` 进行垂直堆叠，并重新生成连续时间轴 (`time_s`)，解决了片段拼接处的时间戳重叠问题。
4. **汇总文件与原始文件的差异化处理**
   探测发现 `Dataset4.mat` 等文件仅包含工况描述字符串而非采样矩阵。Pipeline 增加了**智能探测模块**：若检测到时序缺失，则自动跳过 `timeseries` 生成，仅从 `Capacity` 变量中提取长达 700+ 循环的老化汇总数据。

## 4. 缺失字段说明
受原始数据局限，以下字段为空：
* `internal_resistance_Ohm`: 原始数据未提供每个循环的内阻测量。
* `temperature_C` (部分片段): 部分工况未记录温度，脚本使用 25.0℃ 或 `NaN` 进行填充。
