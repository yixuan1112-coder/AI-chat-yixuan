# SNL Battery Dataset - 数据整理说明 (README)

## 1. 数据集概况

* **Dataset ID**: `dataset_06`
* **原始数据规模**: 包含 15C, 25C, 35C 等多个测试温度批次子文件夹下的多组真实物理电池数据。
* **电池规格**: 18650 商业圆柱形电芯。涵盖三种规格：LFP (1.1Ah, 3.3V), NMC (3.0Ah, 3.6V), NCA (3.2Ah, 3.6V)。
* **材料体系**: 横向对比三种体系（LFP-A123, NMC-LG Chem, NCA-Panasonic），负极均为 Graphite (石墨)。
* **包含文件**:
1. `dataset_06_metadata.csv` (元数据总表，18 列 XJTU 标准)
2. `*_cycle_summary.csv` (各电池周期汇总表，分批次存放在子文件夹)
3. `*_timeseries.csv` (各电池时间序列表，分批次存放在子文件夹)

## 2. 字段映射与物理量标准化

根据 `BatteryTwin Schema v0.2` 及下游 18 列特征宽表规范，对原始数据进行了以下标准化处理：

* **唯一化标识 (Cell ID 防重名机制)**: 由于原始数据分散在 `15C_Test` 等不同批次文件夹中，极易出现同名文件（如 `cell_1.csv`）。为确保合并大表时的全局唯一性，`cell_id` 已采用 `{Dataset_ID}_{Folder_Name}_{File_Name}` 格式拼接（例：`dataset_06_15C_Test_cell_1`）。
* **充放电协议动态映射**: 原始文件名中隐式包含了充放电倍率信息。脚本通过正则表达式提取（如 `_0.5-1C_`），并结合该化学体系的工作电压窗口，自动组装成标准文本（例：`CC-CV 0.5C to 4.2V` 和 `CC 1C to 2.5V`）填入 `charge/discharge_protocol` 字段。
* **数据划分预留**: `split_tag` 字段在 ETL 阶段统一预设为 `unassigned`，预留给后续的 LightGBM 机器学习流水线进行动态划分。

## 3. 核心工程挑战与清洗逻辑记录

*注：SNL 数据集具有极高的物性多样性和目录嵌套复杂度，已通过定制化多模块 Python 脚本解决以下工程痛点：*

1. **子文件夹嵌套导致的文件读取与输出隔离问题**
原始文件并非平铺在根目录，而是存放在不同的温度 Batch 文件夹中。常规的单层 `glob.glob` 搜索无法读取。故在 ETL 脚本中引入了 `os.scandir` 遍历逻辑，让脚本能够穿透子文件夹，并在 `processed` 输出目录中自动复刻对应名称的子文件夹进行结果隔离存放。
2. **多化学体系混杂的元数据提取与补全**
由于同一数据集中混杂了 LFP、NMC 和 NCA 三种截然不同的电池，其额定容量和电压窗口差异巨大（如 LFP 仅 1.1Ah，NCA 达 3.2Ah）。为对齐 18 列标准元数据表，脚本在解析文件名时内置了硬核的“物理字典映射”，确保不仅能抓取材料名称，还能自动精准补全各自对应的容量、电压上限（4.2V/3.6V）与下限（2.5V/2.0V）。
3. **派生特征的跨表精准提取**
汇总表所需的 `charge_duration_s`（充电时长）和 `temperature_max_C`（最高温度）等特征在原始汇总文件中缺失。为此，采用了“双表联动”架构：在处理 summary 的同时加载底层的 timeseries 数据，通过判定电流方向（`Current > 0.01`）和周期分组（`groupby(cycle)`），实现秒级精度的时间积分与温度极值提取。
