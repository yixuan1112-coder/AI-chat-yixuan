# JRC 电池数据集 - 数据处理说明 (README)

## 1. 数据集概览
* **数据集编号 (Dataset ID)**: `dataset_40`
* **原始数据范围**: 单个宽表格式 CSV 文件，包含 6 种不同功率工况（3300W 至 330W）下的高功率动态测试数据。
* **电池规格**: 高能量 NMC 软包电池，额定容量 25Ah。
* **化学体系**: NMC (镍钴锰酸锂)。
* **产出文件**:
  1. `dataset_40_metadata.csv` (元数据表)
  2. `JRC_{Condition}_cycle_summary.csv` (循环级汇总表)
  3. `JRC_{Condition}_timeseries.csv` (高频时序表)

## 2. 字段映射与标准化
严格遵循 `BatteryTwin Schema v0.2` 规范进行标准化：
* **格式规范化**: 原始欧洲格式 CSV 使用分号 (`;`) 作为分隔符，并用逗号 (`,`) 作为小数点。脚本统一将其转换为标准小数点 (`.`)，以确保与 Python 数据栈的兼容性。
* **宽表转长表**: 原始数据将 6 组电池测试并列排布在同一张表。Pipeline 通过列分离逻辑将其拆分为以工况（如 `3300W`）命名的独立文件。
* **电流符号对齐**: 严格执行“放电为负”原则。脚本自动处理原始电流信号，确保放电阶段 `current_A` 为负值。
* **特征推导 (安时积分)**: 由于原始数据未提供容量字段，脚本通过对电流进行**安时积分 (Ah Integration)** 自动计算出放电容量 (`capacity_Ah`)。

## 3. 核心工程挑战与流水线逻辑
*备注：JRC 数据集处理的核心在于宽表列解耦以及从原始电信号中推导物理特征。*

1. **多电池混合宽表的解耦**
   单个原始文件混合了 6 种测试场景。脚本实现了一套列名映射策略，精确提取特定工况的信号对（如 `V_D_3300` 和 `C_D_3300`），并重构为独立的电池记录，防止不同测试条件间的数据交叉。
2. **隐式容量特征重构**
   不同于老化数据集，动态功率工况通常不直接给出循环容量。Pipeline 内置了实时积分模块，通过高频电流信号计算累积放电安时数，从而生成符合 Schema 的 `capacity_Ah` 字段。
3. **时间轴标准化**
   原始时间戳被重置为从 0 开始的连续秒级坐标 (`time_s`)，确保动态功率波形在不同工况下具有统一的时间基准，便于机器学习模型的输入对齐。

## 4. 缺失字段说明
受原始数据局限，以下字段为空：
* `internal_resistance_Ohm`: 动态脉冲测试中未包含显式的内阻测量。
* `SOH` & `RUL`: 该数据属于特性测试（仅 Cycle 1），不涉及长期循环老化，故不适用寿命衰减指标。
